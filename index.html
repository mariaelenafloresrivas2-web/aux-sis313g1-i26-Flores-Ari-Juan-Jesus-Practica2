PRACTICA N2 GIT, GITHUB-HTML

  1. ¿Qué es una rama (branch) en Git y para qué sirve?

    Una rama (branch) en Git es una línea de desarrollo independiente que permite a los desarrolladores trabajar en diferentes características, correcciones de errores o experimentos sin afectar la rama principal (generalmente llamada "main" o "master"). Las ramas facilitan la colaboración y el manejo de cambios, ya que cada rama puede ser fusionada (merge) con la rama principal una vez que el trabajo esté completo y probado.

  2. ¿Cómo se crea y se cambia entre ramas en Git?

    Para crear una nueva rama en Git, se utiliza el comando `git branch `. Para cambiar entre ramas, se utiliza el comando `git checkout `. Alternativamente, en versiones recientes de Git, se puede usar `git switch `.

  3. ¿Cuál es la diferencia entre la rama main y otras ramas?

    La rama main (o master) es la rama principal del repositorio, donde se encuentra el código estable y funcional. Las otras ramas son ramas de desarrollo que permiten trabajar en nuevas características o correcciones sin afectar el código principal.

  4. ¿Cómo se combinan ramas en Git (merge)?

    El comando `git merge` se utiliza para combinar los cambios de una rama con otra. Se ejecuta desde la rama a la que se desea fusionar los cambios, y se especifica el nombre de la rama que contiene los cambios que se quieren integrar.

  5. ¿Qué problemas pueden ocurrir al trabajar con ramas (conflictos) y cómo se resuelven?

    Los conflictos ocurren cuando dos ramas tienen cambios en las mismas líneas de código. Para resolverlos, se debe revisar los archivos afectados, decidir qué cambios conservar y marcar el conflicto como resuelto antes de completar la fusión.
